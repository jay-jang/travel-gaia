> The SIPP Code (Standard Interline Passenger Procedure code) is the legacy GDS-origin name for the four-letter rental-car classification code that ACRISS later expanded and now maintains. The term is still used, especially in North America, when describing car content in GDS environments.

SIPP codes share the same four-position structure as ACRISS codes — Category, Type, Transmission/Drive, and Fuel/Air-conditioning — and in everyday GDS and agency usage 'SIPP code' and 'ACRISS code' refer to the same four-letter vehicle descriptor. The distinction is mainly historical and regional: SIPP is the original interline-era label inherited from passenger procedures, while ACRISS is the modern governing standard that formalized and broadened the value sets. Understanding the SIPP label is essential for reading older GDS car displays, contracts, and documentation that have not adopted the ACRISS naming.

**한국어 / Korean** — **SIPP 코드(SIPP Code)** — SIPP Code(Standard Interline Passenger Procedure code)는 ACRISS가 이후 확장해 현재 관리하는 네 글자 렌터카 분류 코드의 더 오래된 GDS 기원 명칭이다. 특히 북미에서 GDS 환경의 렌터카 콘텐츠를 설명할 때 여전히 사용된다.

SIPP 코드는 ACRISS 코드와 동일한 네 자리 구조(등급, 차종, 변속기·구동, 연료·에어컨)를 공유하며, 일상적인 GDS·여행사 사용에서 'SIPP 코드'와 'ACRISS 코드'는 동일한 네 글자 차량 식별자를 가리킨다. 차이는 주로 역사적·지역적인 것으로, SIPP는 여객 절차에서 물려받은 인터라인 시대의 원래 명칭이고 ACRISS는 값 집합을 공식화·확장한 현대의 관장 표준이다. ACRISS 명칭을 채택하지 않은 구형 GDS 렌터카 화면, 계약, 문서를 읽으려면 SIPP라는 명칭을 이해하는 것이 중요하다.
