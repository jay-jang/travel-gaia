# Travel Gaia 재설계 — Round 2 상호 비평·수렴

라운드1에서 세 자문가(claude code, codex, agy)가 Travel Gaia UI 전면 재설계안을 냈고, **거의 수렴**했다. 아래 합의는 재론하지 말고, **쟁점 6개**에 대해 다른 안을 비평한 뒤 **최종 합의안**을 내라. 제약은 동일(단일 index.html·0의존성·오프라인·327용어/16카테고리·한영·다크모드).

## 이미 합의된 것 (재론 불필요)
- 종이+잉크 에디토리얼 팔레트, 소수 뮤트 악센트, 카테고리색은 스와치/키라인만(큰 채움 금지)
- 에디토리얼 세리프 + 중립 산세리프 + 모노
- 헤어라인 괘선, 그림자 없음, radius 0~2px, 장식적 모션 없음
- **Matuschak 스택 칼럼**(링크 클릭→우측 패널 적층)으로 교차참조 탐색
- **Tufte 우측 마진 사이드노트**(related/distinctions/sources)
- 표제어별 **로컬 ego 미니그래프** + 전체 그래프는 조용한 모노톤 아틀라스
- 색인/카테고리=책형 조밀 표 리스트, **distinctions를 표로** 노출

## 세 안 요약과 쟁점
- **쟁점1 세리프**: claude=Newsreader / codex=Source Serif 4 / agy=Lora(~35KB woff2). 오프라인 임베드+권위+소자(작은 글씨) 가독+한글 폴백 조화를 기준으로 **하나로**.
- **쟁점2 악센트**: claude=단일 oxblood(#7a2520) / codex=도메인별 다중(air #8a4b2a·hotel #2f6f73·std) / agy=단일 navy(#2E5B82)+카테고리 뮤트. **하나로**.
- **쟁점3 기본 브라우징 레이아웃**: codex=3분할(좌 필터/중 목록·본문/우 맥락) 기본 / claude·agy=2단(좌 사이드바 + 우 스택영역). 데스크톱 디폴트를 **하나로**.
- **쟁점4 모션 허용범위**: claude·codex=절제(push, 120~180ms) / agy=슬라이드인 트랜지션+패널별 close/fold 컨트롤. 어디까지 허용?
- **쟁점5 그래프 인터랙션**: hover peek(claude) vs 우측 dossier 패널(codex) vs 하단 고정 인포바(agy); 미니그래프 inline-SVG vs canvas. **하나로**.
- **쟁점6 실무 리스크 완화**: (a) 스택 패널 다수시 DOM/메모리(agy 지적: 닫힌 패널 GC, 깊이 캡?) (b) 데스크톱 가로스크롤 피로(agy: 점프/스크롤 버튼) (c) 폰트 용량(codex/claude) (d) 한글 폰트(claude: 시스템 폴백). 각 항목 **채택 완화책 1줄**.
