# Travel Wiki — 전면 UI 재설계 자문 브리프 (Round 1)

너는 시니어 프로덕트/비주얼 디자이너다. 아래 제품의 UI를 **전면 재설계**하는 제안을 달라.

## 제품
**Travel Wiki** = 여행 산업 용어 백과사전. 항공/호텔/유통/정산/표준 등 **공급자·표준별로 다른 이름을 통일**하는 레퍼런스 위키. 가치의 핵심은 단순 정의가 아니라 **교차 동의어 매핑 + 혼동되는 쌍의 구분(distinctions)**.
- 현재 **327개 표제어 / 16개 카테고리**, 전부 **한/영 양국어**, 항목마다 정의·확장정의·관계(related/broader/narrower/...)·distinctions·출처(sources)·라인아트 **SVG 아이콘** 보유.
- 라우트: 홈, 카테고리, 표제어 상세(`#/term/<id>`), 지식그래프(`#/graph`), 전체 색인(`#/index`, A–Z/카테고리별), 클래스룸(학습 흐름), 검색.

## 절대 제약 (이걸 깨면 안 됨)
1. **단일 `index.html` 파일, 외부 의존성 0, 완전 오프라인**(더블클릭 동작). CSS/JS/JSON/폰트(base64) 전부 인라인. **CDN·외부폰트·외부이미지 금지.**
2. 데이터는 빌드시 인라인되는 JSON. 위 데이터 모델/라우트는 유지.
3. 한/영 토글, 다크모드 토글 유지.

## 문제 (왜 재설계하나)
현재 UI가 **"AI가 자동 생성한 SaaS 랜딩" 티가 강하다**: 보라→자홍 3색 그라데이션 히어로, Space Grotesk 거대 디스플레이 폰트, 글래스모피즘 블러 헤더, 빛나는 그라데이션 로고, 전부 둥근 모서리(14px)+그림자 카드, 숫자 카운트업 애니메이션. 진부하고 신뢰감이 떨어진다.

## 목표
- **"AI 티"를 없애고** 진짜 편집물/레퍼런스북 같은 **권위 있고 차분한** 인상.
- 방대한 **지식 DB를 쉽게 탐색·검색**. 텍스트 내비게이션은 물론 **그래픽(시각적) 내비게이션**으로도 쉽게 돌아다닐 수 있어야 함.
- 전체를 다 재설계.

## 참고 레퍼런스 (인터넷 조사 결과 — 이 방향성 참고)
- **Tufte CSS / Edward Tufte handouts**: 정교하게 짜인 타이포그래피, 본문 옆 **사이드노트(margin notes)**, 본문과 그래픽의 긴밀한 통합, 극도의 절제. → 권위 있는 레퍼런스 느낌.
- **Andy Matuschak's notes (notes.andymatuschak.org)**: **stacked pages/columns** — 링크를 클릭하면 새 칼럼이 우측에 쌓여 **맥락을 잃지 않고** 연결을 따라감. atomic·concept-centric·densely linked. → 교차참조 많은 우리 데이터에 매우 적합한 "그래픽적" 탐색.
- **Distill.pub / Stanford Encyclopedia of Philosophy**: 매우 평이하지만 권위 있는 상호링크 레퍼런스.
- **Linear 헬프센터**: zero-distraction, 콘텐츠 우선, doc형 사이드바.
- **에디토리얼 디자인 정설(2025)**: "**editorial serif + neat grotesque sans**" 조합이 콘텐츠+유틸리티 UI에 적합. 모노크롬/절제 팔레트, 타이포 주도 위계, 의도적 그리드, 세리프=권위/신뢰. 헤어라인(rule) 위주, 카드+그림자 남발 지양.
- 지식그래프 UI 정설: 개념 목록 + 시각 그래프 + 컨트롤, 노드는 타입별 색, 호버시 상세, 그래프 언어 없이 multi-hop 탐색.

## 산출물 형식 (반드시 아래 섹션 순서·라벨로, 간결하게)
```
[CONCEPT] 한 줄 디자인 컨셉/메타포
[VISUAL SYSTEM] 폰트 페어링(구체 폰트명, 임베드 가능한 것), 팔레트(hex 몇 개), 밀도, 보더/그림자 철학, 모션 철학
[IA & NAV] 텍스트 내비게이션 모델 + 지식DB 브라우징 방식 + 그래픽 내비게이션 접근(구체적으로)
[KEY SCREENS] 홈/디렉터리, 표제어 상세, 카테고리, 그래프, 색인 — 각 레이아웃을 말로 묘사
[ANTI-AI] 왜 이 안이 진부한 AI룩을 피하는지 2~3줄
[HTML/CSS] 구체 토큰: CSS 변수(색/타이포/간격), 핵심 레이아웃(grid), 시그니처 CSS 규칙 3~5개
[RISKS] 단일파일·오프라인 제약하 최대 리스크/트레이드오프
```
결론 지향. 일반론 말고 **이 제품에 특정된** 구체안으로.
